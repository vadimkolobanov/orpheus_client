# ADR-0001: Процесс разработки с Cursor (docs + changelog + worklog)

## Контекст
Нужно, чтобы ИИ/агент в Cursor не только писал код, но и:
- обновлял документацию,
- фиксировал изменения в `CHANGELOG.md`,
- вёл журнал действий (`AI_WORKLOG.md`),
- не “забывал” это делать.

## Решение
1. В репозитории существуют обязательные артефакты:
   - `CHANGELOG.md` (Keep a Changelog, секция `Unreleased`)
   - `AI_WORKLOG.md` (хронологический журнал)
   - `docs/` (операционная и архитектурная документация)
2. В `.cursor/rules/` лежат правила проекта, которые требуют обновлять эти артефакты при изменениях.
3. В `.cursor/commands/` лежат команды-шаблоны для типовых действий (обновить changelog, добавить запись worklog, подготовить коммит).
4. Git hook (`.githooks/pre-commit`) блокирует коммит изменений кода, если не обновлены `CHANGELOG.md` и `AI_WORKLOG.md`.

## Последствия
- История изменений и контекст решений становятся воспроизводимыми.
- Чуть больше дисциплины на каждый PR/коммит, но это окупается качеством сопровождения.
