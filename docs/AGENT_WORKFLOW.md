# Работа с Cursor в агентном режиме (как часы)

Этот документ — “контракт” между вами и агентом: как ставить задачи так, чтобы агент делал работу предсказуемо и профессионально.

## 1) Как ставить задачу (рекомендуемый шаблон)

Минимум:
- **Repo**: `orpheus_client` (client) / `Programs/orpheus` (server) / `OPHEUS_ADMIN` (admin)
- **Цель**: что нужно получить
- **Критерии готовности**: как проверить “сделано”

Опционально:
- **Ограничения**: “без коммита”, “только анализ”, “не трогать UI”, “сделай 2 коммита”, “без новых зависимостей”
- **Контекст**: ссылки на файлы/скриншоты/логи

Пример:
```
Repo: orpheus_client
Цель: добавить переключатель X в настройках и сохранить в локальной базе
Готово, когда: есть тест на сохранение, UI отображает состояние после перезапуска
Ограничения: без изменения дизайна кроме добавления пункта
```

## 2) Что агент делает автоматически
- Находит нужные места в коде и делает минимальные изменения.
- Обновляет артефакты процесса при изменениях кода:
  - `CHANGELOG.md` (Unreleased)
  - `AI_WORKLOG.md`
  - `docs/**` (если меняется поведение/инструкции)
- Если вы не запретили — готовит/делает коммит (в клиенте: `.\scripts\auto-commit.ps1`).

## 3) Быстрые флаги (короткие команды в задаче)
- `без коммита` — не делать `git commit`
- `только анализ` — не менять файлы, только вывод/рекомендации
- `одним коммитом` / `двумя коммитами` — явно задать гранулярность
- `только клиент` / `только сервер` / `только админка`

## 4) Чеклист “готово”
- Тесты/анализатор зелёные (если применимо)
- Артефакты обновлены (`CHANGELOG.md`, `AI_WORKLOG.md`)
- Ничего лишнего не затронуто
