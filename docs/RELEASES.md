# Orpheus Client — релизы

## Источники “что нового”
- **Публичные release notes** ведутся в админ‑панели `OPHEUS_ADMIN` → раздел **«Версии»** (`app_versions`). Это то, что видят пользователи.
- `CHANGELOG.md` в этом репозитории — внутренний журнал разработки (для команды).

## Минимальный процесс релиза клиента (чек‑лист)
- Обновить версию/артефакты согласно текущему процессу проекта.
- Прогнать тесты:
  - `flutter test` (или `.\test_runner.ps1` для отчётов)
- Убедиться, что критичные сценарии не регресснули (см. `docs/testing/README.md` и `docs/TESTING_GAPS.md`).
- Обновить публичные release notes в `OPHEUS_ADMIN` (раздел «Версии»).

## Замечания по совместимости
Если меняются сетевые контракты/URL/домены:
- фиксировать решение в ADR (`docs/DECISIONS/`);
- документировать миграцию (например `docs/DOMAIN_MIGRATION_orpheus.click.md`).
