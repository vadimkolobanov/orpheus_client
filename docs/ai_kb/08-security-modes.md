# Безопасность в приложении (PIN / Duress / Wipe)

Ниже — пользовательское описание режимов безопасности: что они делают и где находятся.

## Где включать
Откройте вкладку **Профиль** → **Безопасность**.

## PIN‑код (блокировка приложения)
- Если PIN **не задан** — вход в приложение открыт.
- Если PIN **включён** — приложение может блокироваться при запуске и/или при сворачивании (в зависимости от настроек).

## Duress‑код (код принуждения)
Это “второй PIN”, который открывает приложение в **режиме правдоподобного отрицания**:
- показывается “пустой профиль” (например 0 контактов/сообщений);
- реальные данные **не удаляются**, а скрываются в интерфейсе, пока активен duress‑режим.

## Код удаления (Panic wipe code)
Это отдельный код для **полного удаления данных на устройстве**.
После ввода обычно требуется подтверждение (например удержание кнопки), чтобы избежать случайного стирания.

## Auto‑wipe (удаление после N неверных попыток)
Опция: удалить данные после заданного числа неверных попыток ввода PIN.
Включайте только если понимаете риск необратимой потери данных (ошибка ввода/дети/случайные касания).

## “Жест” panic wipe
Опция (по умолчанию может быть выключена): несколько быстрых уходов приложения в фон → wipe.
Это сделано для экстренных ситуаций, но может быть менее предсказуемым, чем отдельный wipe‑код.

## Экспорт аккаунта (приватный ключ) — очень важно
Вкладка **Профиль** → **Экспорт аккаунта** показывает приватный ключ.

Приватный ключ = полный доступ к аккаунту.
- Не отправляйте его никому (в том числе “в поддержку”).
- Храните только в безопасном месте.

## Защита от скриншотов/записи экрана
На Android приложение включает защиту от скриншотов/записи экрана для приватности.
Для обмена ID используйте “Поделиться контактом” или копирование ключа, а не скриншоты.
