# Приватность и безопасность (коротко и честно)

## Что Orpheus старается обеспечить
- **E2E‑шифрование сообщений**: текст сообщений должен быть доступен только участникам переписки.
- **Privacy‑уведомления**: пуш‑уведомления не должны раскрывать содержимое сообщения (обычно показывается “Новое сообщение”).
- **Защита интерфейса при физическом доступе**: PIN/блокировки/duress‑режимы снижают риск случайного просмотра данных.

## Что Orpheus НЕ может гарантировать
- Защиту данных при **компрометации устройства** (root, вредоносное ПО, forensic‑доступ к файловой системе).
- Полную “анонимность от всего”: как и любой сетевой сервис, инфраструктура может видеть **технические метаданные** (например, время соединений, IP/сетевые параметры).
- Полное физическое уничтожение данных только за счёт “скрытия UI” (duress mode скрывает интерфейс, но не является обещанием “стереть с диска”).

## Что важно пользователю
- **Ваш ID — публичный ключ**. Потеря ключей может означать потерю доступа к идентичности.
- История сообщений хранится **локально на устройстве** ради удобства.
- При обращении в поддержку будьте аккуратны с тем, что отправляете (скриншоты/логи могут содержать лишние данные).
