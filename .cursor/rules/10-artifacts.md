# Обязательные артефакты изменений (docs / changelog / worklog)

При любых изменениях, которые влияют на код/поведение/конфигурацию клиента:

1) Обнови `CHANGELOG.md` → секция `## [Unreleased]`  
2) Добавь запись в `AI_WORKLOG.md` (что/когда/файлы/команды)  
3) Обнови релевантную документацию в `docs/` (если поменялось "как запускать/как пользоваться/как работает")  
4) **ОБЯЗАТЕЛЬНО:** Добавь ВСЕ файлы в git (`git add .`) перед коммитом - не оставляй файлы неотслеживаемыми!

Правила качества:
- Не добавляй секреты/ключи. Если найдёшь — остановись и предложи безопасный план.
- Старайся не трогать лишние файлы; минимальный дифф.
- Если меняешь поведение — добавь/обнови тесты или явно объясни почему тестов нет.
- **ВСЕГДА** используй `git add .` чтобы все новые и измененные файлы попали в git.
