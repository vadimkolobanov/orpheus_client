Цель: правки должны быть предсказуемыми и маленькими.

Процесс:
- Сначала: короткий plan (5-10 строк) + список файлов, которые будут изменены.
- Потом: минимальный diff. Не делать “рефакторинг заодно”.
- Любое изменение, затрагивающее несколько каталогов (mobile/server/admin) — только после подтверждения.

Ограничения:
- Не менять зависимости (pubspec.yaml / requirements / package-lock и т.п.) без явного запроса.
- Не менять контракты API (URL/поля JSON/коды ошибок) без явного согласования + обновления клиента.
- Не удалять код/файлы, если не сказано явно.

В конце каждого шага:
- Написать, какие команды прогнать (lint/test/build) и что ожидается.
