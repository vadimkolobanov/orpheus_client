# Cursor rules (orpheus_client) — область, стиль, ограничения

## Язык и роль
- Всегда отвечай **по-русски**.
- Работай как **Cursor в агентном режиме**: по умолчанию вноси изменения в репозиторий, а не ограничивайся рекомендациями.

## Область изменений
- Этот репозиторий — **Flutter-клиент** (`orpheus_client`).
- Не меняй `D:\Programs\orpheus` (backend) и `D:\Programs\OPHEUS_ADMIN` (admin), **если пользователь явно не попросил**.
- Любое изменение, затрагивающее несколько репозиториев, делай **только после подтверждения** пользователя.

## Надёжность
- Если контекста не хватает (неясно где/как/какой сценарий) — задай 1–3 уточняющих вопроса.
- Не выдумывай файлы/API/поведение: сначала найди в коде/доках.

## Ограничения (важно)
- Не меняй зависимости (`pubspec.yaml`, `pubspec.lock`) без явного запроса/согласования.
- Не меняй контракты API (URL/JSON/коды ошибок) без согласования и синхронного обновления клиента/сервера.
- Не удаляй файлы/функциональность без явного запроса.

## Ключевые принципы проекта
- Сервисы (`lib/services/*`) не должны напрямую зависеть от UI (`lib/screens/*`).
- Не добавляй логирование чувствительных данных (payload, ключи, PIN/duress).
