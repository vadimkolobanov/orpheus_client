# Cursor rules (orpheus_client) — git дисциплина

## Никаких “потерянных” файлов
- После изменений проверь `git status`.
- Если появились новые файлы (tests/docs/assets) — **обязательно добавь их в git**.
- Перед коммитом: `git add .` (если пользователь не попросил иначе).

## Безопасность репозитория
- Не добавляй секреты/ключи/токены в git.
- Если находишь потенциальный секрет в рабочих изменениях — остановись и попроси подтверждение/замену на env/config.
