# Правила работы с файлами и git

## КРИТИЧЕСКИ ВАЖНО: Все файлы должны быть в git

При создании или изменении ЛЮБЫХ файлов:

1. **ВСЕГДА** добавляй файлы в git перед коммитом:
   ```bash
   git add .
   ```
   - Используй `git add .` (не отдельные файлы)
   - Это гарантирует, что все новые и измененные файлы попадут в git
   - НЕ оставляй файлы неотслеживаемыми (красными в IDE)

2. **При создании нового файла:**
   - Сразу после создания файла он должен быть добавлен в git
   - НЕ создавай файлы, которые не попадут в git
   - Все созданные файлы ДОЛЖНЫ быть отслеживаемыми

3. **Проверка перед коммитом:**
   - Убедись, что `git status` не показывает неотслеживаемые файлы (untracked files)
   - Все новые файлы должны быть в staged area (зеленые в IDE)

4. **Исключения (уже в .gitignore):**
   - build/, .dart_tool/, .idea/, *.log и т.д. - эти файлы игнорируются намеренно
   - Но все файлы кода, документации, конфигов ДОЛЖНЫ быть в git

5. **Проблема:** Если файлы не в git, они теряются при работе на другом компьютере!

## Порядок действий при изменениях:

1. Создай/измени файлы
2. Обнови артефакты (CHANGELOG.md, AI_WORKLOG.md)
3. **ОБЯЗАТЕЛЬНО:** `git add .` (добавить ВСЕ файлы)
4. Создай коммит: `git commit -m "<type>(client): <описание>"`

## Пример правильной последовательности:

```bash
# 1. Создал файл
write("docs/NEW_FILE.md", "...")

# 2. Обновил артефакты
# (CHANGELOG.md, AI_WORKLOG.md)

# 3. Добавил ВСЕ файлы в git
git add .

# 4. Создал коммит
git commit -m "docs(client): добавлен новый файл документации"
```
